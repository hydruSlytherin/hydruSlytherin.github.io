处理用户token

1.创建一个对象，存储用户的登录信息

Token 是用户登录成功之后服务端返回的一个身份令牌，在项目中的多个业务中需要使用到：

  • 访问需要授权的 API 接口
  • 校验页面的访问权限
  • …

但是我们只有在第一次用户登录成功之后才能拿到 Token，所以为了能在其它模块中获取到 Token 数据，我们需要把它存储到一个公共的位置，方便随时取用。

将数据存储在vuex中，获取方便且及时响应

2.用mutations修改data数据，定义一个mutations函数

3.在index.vue中调用

this.$store.**commit**('setUser', data.data)

4.解决刷新丢失的问题

window.localStorage.setItem(TOKEN_KEY, JSON.stringify(state.user))

将token备份到本地，以字符串形式存储，所以在state中要把它还原成对象

项目初始化

一、用vue-cli脚手架创建项目

二、上传到git

1.在GitHub上新建仓库

2.在本地仓库目录下打开git bush，直接复制GitHub给出的代码，即可提交到GitHub上

3.上传改动后的代码

三、引入素材和组件库

图标

1.在矢量图库中选择素材

2.加入项目，生成在线链接

3.把css样式粘贴到styles目录下的icon.less文件中

4.在全局样式文件index.less中引入

@import "./icon.less";

vant组件库

1.安装vant

npm i vant

2.在main.js中加载注册vant组件

四、封装请求模块

1.安装axios

npm i axios

2.创建src/utils/request.js

登录

1.创建登录页面index.vue

2.使用vant组件库完成导航条、登录表单、图标、验证码按钮，同时按需修改样式

写样式的原则：将公共样式写到全局（src/styles/index.less），将局部样式写到组件内部。

3.实现登录功能

思路：

  • 注册点击登录的事件

  • 获取表单数据（根据接口要求使用 v-model 绑定）

  • 表单验证

  • 发请求提交

  • 根据请求结果做下一步处理
